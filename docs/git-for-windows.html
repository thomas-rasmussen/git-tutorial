Appendix A — Git BASH

Git for Windows is a set of tools for Windows users. Of main interest, Git for Windows provides a Bash emulation used to run Git from the command line. Using Git Bash, Windows users can use “git” commands exactly like one would do in Linux and Unix environments. Besides the full set of Git core commands, Git Bash is also packaged with additional shell commands that can be useful.

When you open Git Bash, you will see something like this:

user_id@computer_id MINGW64 ~
$

The user_id@computer_id is an ID based on name of the user that is logged in, and the name of the device running Git Bash. The MINGW64 part is the name of the compiler used to build Git Bash. The ~ at the end of the first line indicates the current directory. A tilde means that the current directory is the home directory of the user.

A.1 Shortcut keys

An extensive list of available commands in Bash can be found here. Not all of them are available in Git Bash. Below is a short list of useful shortcut keys.

Go to beginning of line (Home) Ctrl+a
Go to end of line (End) Ctrl+e
Previous command (Up arrow) Ctrl+p
Next command (Down arrow) Ctrl+n
Go left/back one word Alt+b
Go right/forward one word Alt+f
Go forward one character Ctrl+f
Go backward one character Ctrl+b
Editing
Copy Ctrl+ins
Paste Shift+ins
Clear the screen Ctrl+l
Delete the character before the cursor (Backspace) Ctrl+h
Delete the character after the cursor Ctrl+d
Process control
Interrupt/kill current process (SIGINT) Ctrl+c

Windows users are used to using Ctrl+cCtrl+c and Ctrl+vCtrl+v to “Copy” and “Paste”, but these shortcuts are not available in Git Bash, since they are reserved for other commands. it is possible to enable “Copy” using Ctrl+Shift+cCtrl+Shift+c and “Paste” using Ctrl+Shift+vCtrl+Shift+v, by right-clicking in the terminal, navigating to Options>Keys, and checking the “Ctrl+Shift+letter shortcuts” box.

A.2 Git BASH commands

It is unfortunately not straightforward to get an overview of all available commands in Git Bash. The full list of available commands can be assessed from files in the /usr/bin directory of the Git Bash installation. Documentation of Linux shell commands in general, and how to use them, can be found with some google-fu. For example, here.

Here we give a (not necessarily detailed) description of some of the available shell commands in Git Bash.

Command Description
cd Change working directory
clear Clear terminal window
echo Display a line of text
ls List directory content
mkdir Make directories
pwd Print working dierectory
rm Remove files
rmdir Remove empty directories
start . Open current directory in explorer
touch Change file timestamps

A.2.1 cd

Syntax: cd [dir]
Changes the current directory to dir.

The default dir value (~) is the value of the $HOME shell variable, the home directory of the user.

Note that forward slashes (/) are used to specify file paths, and that paths can be both on the form C:/path/to/directory and /c/path/to/directory

If the file path contains spaces, enclose the file path in (double) quotes.

Git for Windows also comes with shell integration, so an alternative to using cd we can change the working directory by navigating to the folder in Windows explorer, right click on the folder (shift+right click on Windows 11) and select “Open Git Bash here”.

A.2.1.1 Example - Change directory

We can use cd to change the working directory to a specific folder by providing a path to the folder

user_id@computer_id MINGW64 ~
$ cd C:/path/to/directory
user_id@computer_id MINGW64 /c/path/to/directory
$

Note that the working directory was set as C:/path/to/directory, but is displayed on the form /c/path/to/directory

A.2.1.2 Example - file path with spaces

If a file path contains spaces, enclose the file path in (double) quotes

user_id@computer_id MINGW64 ~
$ cd "/c/path/to/some directory"
user_id@computer_id MINGW64 /c/path/to/some directory
$

A.2.1.3 Example - move to sub directory

To change the current working directory to a subfolder of the current working directory, we can simple set dir to the subfolder name

user_id@computer_id MINGW64 /c/path/to/dir
$ cd subfolder"
user_id@computer_id MINGW64 /c/path/to/dir/subfolder
$

A.2.1.4 Example - move to parent directory

We can move the working directory to the parent directory by setting dir to “..”

user_id@computer_id MINGW64 /c/path/to/dir
$ cd ..
user_id@computer_id MINGW64 /c/path/to
$

You can also use “..” more generally when specifying a file path, to denote the parent directory

user_id@computer_id MINGW64 /c/path/to/dir_a
$ cd ../dir_b
user_id@computer_id MINGW64 /c/path/to/dir_b
$

A.2.1.5 Example - Move to previous directory

We can change the working directory to the previous working directory by setting dir to “-” ( “-” is converted to $OLDPWD).

user_id@computer_id MINGW64 /c/current/cd/dir
$ cd -
user_id@computer_id MINGW64 /c/previous/cd/dir
$

A.2.1.6 Example - Move working directory to user’s home directory

As mentioned earlier, “~” denotes the user’s home directory. We can set current working directory by setting dir to “~”, or simply by not specifying it at all.

user_id@computer_id MINGW64 /c/current/dir
$ cd ~
user_id@computer_id MINGW64 ~
$ cd -
user_id@computer_id MINGW64 /c/current/dir
$ cd
user_id@computer_id MINGW64 ~

A.2.2 clear

Syntax: clear
Clear the terminal screen.

A.2.2.1 Example

Imagine we had the history shown in the terminal below, ending with a clear command.

user_id@computer_id MINGW64 ~
$ cd C:/path/to/directory
user_id@computer_id MINGW64 /c/path/to/directory
$ clear

This would result in the terminal shown below.

user_id@computer_id MINGW64 /c/path/to/directory
$

A.2.4 ls

Syntax: ls [OPTIONS]... [FILE]...

List information about the FILE’s (in the current directory by default). Sorts entries alphabetically by default.

Curated options
-a, –all
Do not ignore entries starting with dot (.), i.e. hidden files.
-1
List one file per line -R, –recursive
List subdirectories recursively.
